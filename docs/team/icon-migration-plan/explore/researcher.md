# Exploration: lucide-react Usage and Internal Icon Catalog

Date: 2026-04-02

## Research question

What lucide-react icons are used across the codebase, what internal icon equivalents exist, and where are the gaps that would need to be filled before lucide-react can be removed?

## Scope

**In scope:**
- All `lucide-react` imports in `packages/ui/src/` (component source and stories)
- All `lucide-react` imports in `packages/demo/src/`
- The internal icon catalog at `packages/ui/src/components/icons/`
- Dependency declarations in `package.json` files
- Icon-to-icon name mapping between lucide and internal catalog

**Out of scope:**
- Visual comparison of icon styles (lucide vs Acronis design language)
- Bundle size analysis
- Migration implementation strategy

---

## Findings

### 1. Internal icon catalog

**Confidence:** Corroborated

The internal icon system lives at `packages/ui/src/components/icons/` and has four layers:

#### 1.1 SVG source files

`packages/ui/src/components/icons/svg/` contains **1,561 SVG files**. File naming convention is `{name}--{size}.svg` (e.g., `chevron-down--16.svg`, `shield--32.svg`). Sizes are 16, 24, 32, and 72.

#### 1.2 BaseIcon component

`base-icon.tsx` exports two wrapper components:

- `BaseIcon` — wraps raw SVG children in a consistent `<svg>` element with `className="size-4 shrink-0"`, `viewBox="0 0 16 16"`, `fill="none"`. Accepts standard SVG props.
- `Icon` — alternative wrapper that accepts a `path` prop (string or ReactNode).

Both use `cn()` for className merging. Default viewBox is 16x16.

#### 1.3 Auto-generated icon components

`auto-generated.tsx` is generated by `npm run generate:icons`. It imports SVGs as React components via `?react` suffix (Vite SVG plugin) and wraps each in `BaseIcon`. This file exports **~830 named icon components** (e.g., `ChevronDownIcon`, `ShieldIcon`, `SearchIcon`, `CheckIcon`).

Each component follows this pattern:
```tsx
export function ChevronDownIcon({ className, ...props }: IconProps) {
  return <BaseIcon className={className} {...props}><ChevronDownSvg /></BaseIcon>
}
```

Multiple size variants are consolidated under a single component name (typically the largest or default size). Some icons have explicit size-suffixed variants when both 16px and 32px versions are exported.

#### 1.4 Dynamic icon loading

`dynamic-icons.tsx` provides runtime icon resolution:
- `getIcon(name)` — returns a component by SVG filename
- `getIconNames()` — returns all available icon names
- `DynamicIcon` — renders by name prop
- `LazyIcon` — lazy-loads icons on demand
- `Icons` — a `Record<string, Component>` map of all icons

#### 1.5 Hand-crafted icons in icon-library.tsx

`icon-library.tsx` exports 7 hand-crafted icons with inline SVG paths: `InfoIcon`, `SuccessIcon`, `WarningIcon`, `CriticalIcon`, `DangerIcon`, `CloseIcon`, `ChevronDownIcon`. These are status/severity icons with hardcoded colors (not using `currentColor` for status icons).

#### 1.6 Categories

`categories.json` organizes icons by prefix-based categories (e.g., "acronis", "add", "ai", "agent"). This is a flat grouping by name prefix, not a semantic categorization.

#### 1.7 Public API

`index.ts` exports everything from `base-icon.tsx`, `dynamic-icons.tsx`, and `auto-generated.tsx`. The `icon-library.tsx` export is commented out. lucide-react is **not** re-exported from the UI package's public API (`packages/ui/src/index.ts`).

## Key takeaways

- The internal icon catalog is large (1,561 SVGs, ~830 exported components) and covers most Acronis business-domain icons, but lacks generic UI primitives common in lucide (text formatting, theme toggles, some navigation metaphors). (Corroborated)
- lucide-react is a **direct runtime dependency** of `packages/ui`, not a peer dependency. Every consumer of the UI package bundles it. (Corroborated)
- The UI component source uses **26 unique lucide icons** across 20 component files. Of these, **2 have no internal equivalent** (`Moon`, `Sun`) and **5 are partial matches** needing visual verification. The remaining 19 have direct internal counterparts. (Corroborated)
- The demo app uses **~145 unique lucide icons**. Many are decorative/illustrative and have no business-domain equivalent in the internal catalog. (Corroborated)
- lucide-react is **not** re-exported from the UI package's public API. Consumers import it directly alongside the UI package. (Corroborated)

## Open questions

1. **Visual parity** — Do the internal icon equivalents (e.g., `ChevronDownIcon`, `SearchIcon`) match the lucide visual style closely enough for a seamless swap, or will components look different?
2. **viewBox mismatch** — Internal icons default to `0 0 16 16` while lucide uses `0 0 24 24`. How should sizing be handled during migration?
3. **currentColor usage** — Internal auto-generated icons inherit color from their SVGs. Do all internal SVGs use `currentColor` for stroke/fill, or do some have hardcoded colors (like the hand-crafted status icons in `icon-library.tsx`)?
4. **Moon/Sun gap** — Should these be added to the internal catalog, or should `mode-toggle.tsx` be treated as a special case?
5. **Demo migration scope** — Should the demo also be migrated off lucide-react, or only the UI component source? The demo's 145 icons represent significant migration effort.
6. **strokeWidth consistency** — lucide icons default to `strokeWidth={2}`. Internal icons may have different stroke widths. This affects visual weight.
